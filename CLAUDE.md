# 音乐人格测试项目

## 项目概述
高审美的"音乐人格测试"H5网页，通过8-12道测试题测出用户的"天命音符"人格（C-B七个音符），重点优化了截图分享体验和移动端表现。

## 技术栈
- **框架**: React 19 + TypeScript
- **构建**: Vite 8.0
- **样式**: Tailwind CSS 3.4
- **部署**: 静态站点部署

## 设计原则

### 视觉风格
- **色彩**: 深黑背景(#06070a) + 米白文字(#f2ede3) + 暖金重点色(#c7b08b)
- **字体**: Baskerville 英文 + Noto Serif SC 中文，营造高级感
- **效果**: 玻璃拟态、微妙发光、克制动画，避免过度装饰

### 交互体验
- **响应式**: 移动端优先，优化点击区域和滑动体验
- **动画**: 使用 CSS 动画，duration 300-700ms，ease-out缓动
- **反馈**: hover、focus、active 状态完整，支持键盘导航

### 分享优化
- **ResultCard**: 专门优化的分享卡片，适合截图保存
- **色彩主题**: 每个音符有专属的发光色彩和视觉识别
- **信息层次**: 清晰的视觉层级，突出"天命音符"概念

## 核心组件说明

### ResultCard 组件
最重要的分享卡片组件，特性：
- 每个音符有专属色彩主题和发光效果
- 优化的布局适合移动端截图
- 微妙的动画增强视觉吸引力
- 包含品牌水印便于传播

### 人格类型系统
7个音符对应7种人格类型：
- **C**: 定序和声者 - 稳定共振
- **D**: 游移追风者 - 流动不愁停滞  
- **E**: 灼光表达者 - 情绪锋面
- **F**: 静夜潜流者 - 深夜质感
- **G**: 结构造梦者 - 理性诗意
- **A**: 月影慢拍者 - 温柔感应
- **B**: 失真边界客 - 边界试探

## 开发规范

### 命名规范
- 组件: PascalCase (如 `ResultCard`)
- 文件: PascalCase for components, camelCase for others
- CSS类: Tailwind utilities + BEM for custom classes

### 性能优化
- 使用 CSS动画而非JS动画
- 组件懒加载和代码分割
- 图片优化和延迟加载
- 避免不必要的重渲染

### 验证命令
```bash
npm run dev      # 开发服务器
npm run build    # 生产构建
npm run preview  # 预览生产版本
npm run lint     # 代码检查
```
